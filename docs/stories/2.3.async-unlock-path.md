# Story
**As a** 网关工程师
**I want** 建立 UNLOCK_REQUIRED 的异步解锁路径（冷路径 KMS）
**so that** DEK 失效时不阻塞热路径，系统可自愈。

# Acceptance Criteria
1. 网关收到 UNLOCK_REQUIRED 后异步解锁，客户端重试间隔 50–200ms（抖动）
2. 解锁失败具备重试退避与告警；无级联超时
3. 指标：`unlock_bg_rate`, `unlock_fail_total`, `unlock_latency_ms`
